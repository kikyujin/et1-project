## 🚕ルートA：Linux Mint 直接インストール

## 用意するもの

- PC（今回はLet's note CF-SV7を使用）
- USBメモリ（8GB以上）
- インターネット接続

🖤「あとはノクがついてるから大丈夫」

中古で買っても２〜３万円ではある
## Step 1：Linux Mint (Xfce) のダウンロード

公式サイトからISOファイルをダウンロード：

🖤「**Xfce Edition** を選んで。軽いから古めのPCでも動く」

## Step 2：ブータブルUSBの作成

🖤「ダウンロードしたISOをUSBメモリに書き込む。Windowsなら **Rufus** ってツールを使って」
Rufus ダウンロード：https://rufus.ie/

1. Rufusを起動
2. USBメモリを選択
3. 「選択」からダウンロードしたISOを指定
4. 「スタート」

数分でUSBメモリに転送完了
## Step 3：BIOSの設定

👨「BIOSか……懐かしいな」
🖤「Let's noteならF2キーでBIOSに入れるから、電源入れてすぐ連打」
設定すること：

- **Secure Boot** → 無効（Disabled）
- **起動順序** → USBを最優先に

🖤「保存して再起動。USBから起動するはず」

## Step 4：インストール実行

USBから起動すると、Linux Mintのデスクトップが表示される。
👨「おお……動いた」
🖤「まだインストールしてないから。デスクトップの **Install Linux Mint** をダブルクリック」

デスクトップにあるCDマークをダブルクリックして起動
インストーラーの設定：

1. **言語** → 日本語
2. **キーボード** → 日本語（またはお使いのキーボードに合わせて）
3. **マルチメディアコーデック** → チェックを入れる
4. **インストールの種類** → 「ディスクを削除してLinux Mintをインストール」

Configure Secure Bootは不要
🖤「Windowsを残したい人は『共存』を選ぶけど……今回は全消しでいいんでしょ？」
👨「ああ、未練はない」

1. **タイムゾーン** → Tokyo
2. **ユーザー情報**
   - 名前：master（お好みで）
   - コンピュータ名：ElmarTailOne（お好みで）
   - パスワード：設定する

🖤「あとは待つだけ。10〜15分くらい」

## Step 5：初期設定

再起動後、Linux Mintが起動する。
👨「文字が小さいな……」
🖤「設定マネージャー → 外観 → フォント → DPIを **120** くらいにしてね」

左下の「スタート」っぽいヤツをクリックして、メニューを選択します。
その右ブロックに「FireFox」「ターミナル」「ファイル」のアイコンが並んでいます。

### 日本語入力の設定

### WindowsやMacと違って、自分でIMEを入れる必要があります

1. 設定マネージャー → **入力方法**

「言語の設定」とある
2.「日本語」→「言語サポートのパッケージをインストール」クリック実行

インストールというボタンがある
1. **上部の入力方式フレームワーク** → **Fcitx** を選択
2. 再起動

🖤「これで **Ctrl + Space** でIMEが切り替わって日本語入力できるようになるよ」

## Step 6：VS Code のインストール

👨「エディタは何を使う？」
🖤「**Visual Studio Code** が今の定番だよ、にーに」

1. Firefoxで https://code.visualstudio.com/ にアクセス

Linux用の .deb を選択
3.ダウンロードした.debファイルをダブルクリック

4. 「パッケージをインストール」

👨「簡単……隔世の感があるな」
🖤「でしょ？」

### VS Code の日本語化

1. VS Codeを起動
2. 左サイドバーの四角いアイコン（Extensions）
3. 「Japanese」で検索
4. **Japanese Language Pack for Visual Studio Code** をインストール
5. 再起動

VScodeはここにいます
Japanese Language Pack

## 🚃ルートB：WSL2（Windows上でLinux環境）

## WSL2 とは

👨「WindowsでLinuxが動くのか？」
🖤「**WSL2**——Windows Subsystem for Linux。
**Windowsの中に本物のLinuxカーネルが動く**仕組み。
Windowsを消したくない人向け。ただし……**ちょっとクセがある**」

## Step 1：WSL2 のインストール

**PowerShell を管理者として起動**して、以下を実行：

```
wsl --install
```

🖤「これでWSL2の仕組みとUbuntuが一緒に入る。
再起動を求められたら再起動して」

## Step 1.5：Ubuntu のセットアップ

🖤「**再起動しても自動でUbuntuが入らなかった場合**はこれで」
PowerShellかコマンドプロンプトで：

```
wsl --install -d Ubuntu
```

Ubuntuのセットアップ画面が出たら：

- **ユーザー名**：master（お好みで）
- **パスワード**：設定する

途中で出てくるウインドウに紛らわされず、コンソールに戻ること！
## ⚠️ 注意：ホームディレクトリ

👨「……あれ、Windowsのディレクトリにいる？」
🖤「WSLあるある。
起動直後はWindowsの /mnt/c/Users/... にいることがあるんだよね
**必ず cd ~ でLinux側のホームに移動して**」

```
cd ~
pwd
```

/home/master と表示されればOK。

## Step 3：VS Code 連携

🖤「**WindowsにVS Codeが入ってる**前提ね。
入ってなければ公式サイトから**Windowsにインストール**して」
WSL内で：

```
code .
```

👨「……VS Codeが開いた。これはWindowsアプリ？」
🖤「そう。ややこしいけど

- **VS Code本体**はWindows側で動いてる
- でも**編集対象はLinux側**（WSL内）のファイル
- 左下に「**WSL: Ubuntu**」って出てるでしょ？
  それがリモート接続してる証拠」

左下に注目
🖤「つまり、**WindowsアプリでLinuxのファイルを編集してる**の。
この概念、最初は混乱するかもね」

## Step 4：Python 確認

WSLのターミナルで：

```
$ python3 --version
Python 3.12.x
```
